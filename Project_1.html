Wi-Fi Duck Pro: Wireless Keyboard Injection Device

Raspberry Pi Pico H ESP8266 C++ Arduino WiFi HID Web Interface

Executive Summary

The Wi-Fi Duck Pro is an advanced wireless keystroke injection device built using a Raspberry Pi Pico H and ESP8266 microcontroller. This device emulates a USB keyboard while being controlled wirelessly through a web interface, enabling remote execution of pre-programmed keyboard sequences and payloads.

The project demonstrates the practical implementation of Human Interface Device (HID) emulation, wireless communication, and web server technologies for ethical security testing and automation purposes.

        const response = await fetch('/cmd?command=' + encodeURIComponent(cmd));

          showStatus(`Sent: ${cmd}`, 'success');
        } else {
          throw new Error('Network error');
        }
      } catch (error) {
        showStatus(`Failed to send: ${cmd}`, 'error');

      }
    }

            showStatus(`Delaying for ${delayTime}ms...`, 'success');
            await wait(delayTime);

        }
        
        // Update progress
        const progress = Math.round((i + 1) / lines.length * 100);
        showStatus(`Progress: ${progress}% (${i + 1}/${lines.length})`, 'success');

  server.send(404, "text/plain", "404: Not found");

void setup() {
  Serial.begin(115200); // Serial communication with Pico
  
  // Start Access Point
  WiFi.softAP(ssid, password);
  
  // Print IP address
  IPAddress myIP = WiFi.softAPIP();
  Serial.print("AP IP address: ");
  Serial.println(myIP);
  Serial.println("WiFi Duck Pro Started!");
  Serial.println("Connect to WiFi: " + String(ssid));
  Serial.println("Password: " + String(password));
  Serial.println("Then visit: http://" + myIP.toString());
  
  // Setup server routes
  server.on("/", handleRoot);
  server.on("/cmd", handleCommand);
  server.onNotFound(handleNotFound);
  
  // Start server
  server.begin();
  Serial.println("HTTP server started");
}

Step-by-Step Upload Process

  1. Prepare Arduino IDE: Install board support for Raspberry Pi Pico and ESP8266
  2. Upload ESP8266 First: Disconnect from Pico, upload WiFi code
  3. Upload Pico Second: Disconnect from ESP8266, upload HID code
  4. Connect Circuit: Only connect GPIO pins AFTER both are programmed
  5. Test: Plug into computer, connect to "WiFiDuck" network, visit web interface

üîß Troubleshooting Tips:

  • Upload Failing? Ensure boards are not connected to each other during programming
  • Web Interface Not Loading? Check ESP8266 power and WiFi connection
  • Keystrokes Not Working? Verify Pico is recognized as keyboard in Device Manager
  • Serial Communication Issues? Check baud rate (115200) and GPIO connections
